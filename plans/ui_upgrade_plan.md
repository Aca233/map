# HOI4 风格 UI 升级计划

## 目标
将现有的基础 UI 升级为类似《钢铁雄心4》(HOI4) 的经典布局，包含顶部资源栏、左上角国家信息、右下角地图模式切换等元素。

## 架构设计

### 1. 整体布局 (Layout)
UI 将采用绝对定位覆盖在 3D 地图之上，主要分为以下几个区域：
- **Top Bar (顶部栏)**: 显示国家资源（政治点数、稳定度、战争支持度、人力、工厂等）、当前时间、游戏速度控制。
- **Top Left (左上角)**: 国家国旗/头像，点击可打开国家概况面板。
- **Left Panel (左侧面板)**: 预留给各种管理界面（政治、科研、外交、生产等），默认隐藏，点击左上角或快捷键呼出。
- **Bottom Right (右下角)**: 详细的地图模式切换按钮组（政治、地形、补给、海域、空域等），以及小地图（可选）。
- **Bottom (底部)**: 预留给部队控制栏、通知消息等。
- **Right (右侧)**: 现有的地块/省份信息面板，优化样式以匹配整体风格。

### 2. 样式风格 (Styling)
- **颜色主题**: 采用深色背景（如深灰、深蓝），带有金属质感或磨砂玻璃效果（`backdrop-filter: blur`）。
- **边框**: 细边框，带有轻微的高光和阴影，模拟立体感。
- **字体**: 清晰的无衬线字体，关键数据使用亮色（如白色、亮绿、亮黄）突出显示。
- **图标**: 广泛使用 SVG 图标来表示资源、面板分类等。

### 3. 组件划分 (Components)
为了保持代码可维护性，建议将 UI 拆分为多个独立的模块或类（如果使用原生 JS/TS），或者使用前端框架（如 React/Vue，但目前项目似乎是原生 TS + Vite）。
由于目前是原生 TS，我们将在 `src/ui/` 目录下创建或更新以下文件：
- `UIManager.ts`: 核心管理器，负责初始化和协调各个 UI 组件。
- `TopBar.ts`: 顶部资源和时间栏。
- `CountryPanel.ts`: 左上角国家信息和左侧主面板。
- `MapModePanel.ts`: 右下角地图模式切换（重构现有的 `#map-mode-bar`）。
- `ProvincePanel.ts`: 现有的地块信息面板（重构和美化）。

## 实施步骤

1. **准备工作**: 收集或生成必要的 UI 图标（SVG 格式），如政治点数、人力、工厂等。
2. **HTML 结构更新**: 修改 `index.html`，添加新的 UI 容器结构。
3. **CSS 样式编写**: 在 `index.html` 或单独的 CSS 文件中编写 HOI4 风格的样式。
4. **TS 逻辑重构**:
   - 更新 `UIManager.ts` 以支持新的布局。
   - 实现顶部栏的数据绑定（模拟数据或连接到游戏状态）。
   - 重构右下角地图模式切换。
   - 美化现有的地块信息面板。
5. **交互完善**: 添加面板的展开/收起动画，按钮的悬浮效果等。

## 状态管理 (GameStateManager)
需要一个状态管理器来提供 UI 所需的数据（如资源、时间）。目前项目中似乎有 `src/game/GameStateManager.ts`，需要检查其内容并与 UI 对接。
